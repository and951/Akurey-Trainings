Para poder definir el tamaño de los cuadrados hay 2 principales posibilidades.

  1. Cotent Box  El valor predeterminado. Cuando el tamaño de una caja, relleno y borde se agregan.
  2. Border Box Cuando el tamaño del rectángulo su relleno y borde se doblan

Se puede notar como en el ejemplo anterior el border box tiene exactamente el tamaño dicho en el css con el height y width , sin embargo en el content-box se ve más grande ya que hay que agregarle el padding y el border.Content =  7.5em = 5 + (2 * 1) + (2 * .25).

Uno de los beneficios de usar border-box es que se puede establecer un height y un width de unidades mixtas sin crear los rectángulos con un tamaño extraño. Un uso fantástico para esto es crear entradas flexibles con un padding-size fijo.

Layout

En lo que respecta con el layout hay tres principales tipos para organizar el display

  • Inline El width y el height son tomados por el contenido y no se le puede especificar de otra manera.
  • Block El width es determinado por el padre y el height es determinado por el contenido. Sin embargo se le pude especificar un height o un width determinado.
  • Inline-block Es lo mismo que el block solo que el height es determinado por el contenido.

Cabe mencionar que el inline-block es ideal para cuando se quiere utilizar un horizontal scrolling.

Positioning
Se dice que un elemento esta posicionado si su valor de posicionamiento tiene algún valor diferente a static. 

Además de mover los elementos tiene la capacidad de cambiar su tamaño. Por ejemplo con las posiciones absolutas y fixed uno puede asignarle posicionamiento de top y bottom a la vez para dejar su tamaño fijo, ósea el height es ignorado.

El posicionamiento puede tomar cualquiera de los siguientes valores.

  • Static El que tiene el elemento por defecto, los valores top,bottom,left y right son ignorados.
  • Fixed El elemento con esta propiedad sera removido del layout y sera re colocado en su posición relativa de la ventana.
  • Absolute El elemento con esta propiedad sera removido del layout y sera re posicionado lo más cerca de su elemento padre tomando en cuenta los valores  top,bottom,left y right.
  • Relative A diferencia de la fixed o la absoluta esquema si se queda en su respectivo layout pero dicho elemento se ve empujado por  los valores  top,bottom,left y right.

El que sea removido de su posición implica que solo los elementos hijos se van a ver afectados por el posicionamiento, por otro lado el static y el relative si afectan a los elementos vecinos. Cabe mencionar que en la relativa cuando se utilizan los valores top, bottom, right o left los vecinos no se ven afectados.

Tables 

El layout tiene diferentes campos donde se utiliza, en el caso de los tables es como el navegador va a administrar los tamaños de las filas y las columnas. En los tables se maneja un algoritmo complejo para su colocación tanto horizontal como vertical, este se trata con el table layout que pude tener dos opciones:

  • Auto    : Este el que esta por defecto y maneja el tamaño de las columnas de uno al otro partiendo de la row más grande, a no ser que especifique un width y un height utiliza para hacer comparaciones relativas.
  • Fixed   : Este intenta de repartir el tamaño de las filas de manera equitativa, a menos de que se le de un tamaño especifico. En este caso intenta de crearlas del tamaño especifico lo mejor posible.

Tabulado de tables

Los tables son utilizados para mostrar conjuntos de datos que se reflejan como matrices. Para cuando se trate con un Table es importante tener en cuenta las siguientes cuestiones. 

  • Los tables de gran tamaño deben de ser atigrado (ósea de un color de promedio en las filas ) o usar un: hover color de fondo (o similar) para ayudar a que el ojo asociar células-de-la-misma-fila.

        


  • Columnas de datos numéricos deben ser alineados a la derecha de modo que los dígitos se alineen.
  • Columnas de la derecha pueden ser necesario que se deban alinear a la derecha para evitar un borde derecho desigual (creo que text-align: justify).
  • Cuando sea posible, el height de las filas debe ser idéntico para hacer más fácil la exploración vertical. (Este principio es conocido en el negocio como "ritmo vertical", y es muy importante).


Es importante no utilizar las tables para otra cosa que no sea tabulado de datos o centrado vertical. Hay muchas razones por las cuales no se recomienda entre estas :

  • Celdas de tabla no respetan desbordamientos
  • Celdas de tabla no respetan posición relativa (Firefox)



Color 

Los colores son mucho más importantes de los que se puede imaginar. Al crear una aplicación su correcta utilización puede definir el éxito del diseño. Hay que tener en cuenta ciertas cosas cuando se eligen los colores de la aplicación.

  • Elegir colores apropiados para el tipo (y antecedentes de tipo)
  • Utilizando skeuomorfismo proceda a ayudar a las personas a reconocer los objetos como lo harían en el mundo real
  • Con contraste para aumentar la legibilidad (nadie puede leer blanco sobre color beige)
  • Uso de patrones como un respaldo para los usuarios daltónicos cuando sea necesario

Los colores se pueden utilizar de diversas maneras entre esas se encuentran:

  • Para expresar emociones. Por ejemplo el rojo tiene la propiedad de que se asocie con comer y por lo tanto puede dar hambre. 
  • Por otro lado se puede utilizar para enfatizar o llamar la atención de ciertos objetos en la pantalla.
